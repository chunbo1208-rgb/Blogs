On Writing in Silence

Chunbo Liu Reflection 2026-03-28

I started this blog with a deliberate constraint: no framework, no database, no comment section, no analytics dashboard. Every post is a file. Every file is a page. The whole thing fits in a folder.

This is not minimalism as aesthetic posture. It is minimalism as necessity — because the moment you add infrastructure, you start maintaining infrastructure. And maintenance is the enemy of writing.

"The best tool for writing is one that gets out of the way."

The Problem with Platforms

I've written on Medium, on Notion, on Ghost, on Substack. Each platform eventually becomes a product decision someone else made. The font changes. The revenue model shifts. The export format is lossy. You are a tenant, not an owner.

With plain HTML, there is no platform to lose. There is only a file, and a browser that will render it faithfully for the next thirty years.

Why Write at All

Writing is not documentation. It is not "sharing" in the social media sense. It is a technology for thinking — specifically, for discovering what you actually believe as opposed to what you imagine you believe.

Most of my beliefs, before I write them down, are incoherent. They are vibes, not arguments. The act of writing forces each claim into a sentence, each sentence into a paragraph, each paragraph into a position that could be wrong. This is uncomfortable. It is also the only useful thing.

Silence as Feature

There are no comments here. No likes, no shares, no view counts visible to anyone but me. This is intentional. Feedback loops on public writing tend to optimize for the wrong things: controversy, relatability, simplicity, affect.

Writing without an audience changes what you write. It makes you honest about whether the sentence is actually good, rather than whether it will be well-received.

"Write for yourself. If others find it useful, that is a bonus."

I don't know if this blog will ever have readers. I write it anyway, because the alternative — not writing — is worse. The thinking doesn't happen without the writing. The files accumulate quietly. That is enough.
